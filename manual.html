1 Requisitos

Lo que necesitás para clonar y usar la aplicación.

Flutter 3.44.x (stable) Git Google Chrome (modo web) Cuenta Firebase (solo para la nube)

La app funciona sin configuración: los datos se guardan localmente en el dispositivo aunque no uses Firebase.

2 Instalar y ejecutar (paso a paso)

Cuatro pasos y la app corre en tu máquina.

  1. Clonar el repositorio

    Descargá el código desde GitHub.

    git clone https://github.com/Iedimes/AgroResearchPro.git
cd AgroResearchPro

  2. Instalar dependencias

    Descarga los paquetes de Flutter.

    flutter pub get

  3. Elegir destino y ejecutar

    Web (la más simple en escritorio), Android o Windows.

    flutter run -d chrome      # web
flutter run -d android    # dispositivo/emulador
flutter run -d windows    # escritorio

  4. ¡Listo!

    Abrí la app, tocá + para crear registros y explorá el Panel de Investigación.

3 Configurar la nube (Firebase)

La configuración ya viene en el repo; solo habilitás los servicios en consola.

La sincronización requiere Authentication anónimo y una base de Firestore. Sin esto, la app igual funciona, pero solo guarda en el dispositivo.

  1. Authentication → Sign-in method → Anonymous

    Habilitarlo en el proyecto agroresearch-pro.

  2. Firestore Database → Create database

    Crear la base (producción o test).

  3. Desplegar las reglas

    Desde la carpeta del proyecto, con la Firebase CLI:

    firebase use agroresearch-pro
firebase deploy --only firestore

    Las reglas incluidas permiten leer/escribir con sesión autenticada.

4 Módulos de la app

Cada lista tiene un botón + para crear y permite tocar un registro para editarlo. Todos muestran su estado de sincronización.

🌱

Gestión de Ensayos

Nombre, cultivo (Soja, Trigo, Maíz, Sorgo), responsable, repeticiones, parcelas, GPS y lote.

🐛

Evaluación de Enfermedades

Ensayo, cultivo, fecha, enfermedad, severidad % e incidencia %, parcela y notas.

🧪

Control Experimental

Barra experimental, fungicida, fertilizante u otro; producto, dosis, volumen de caldo, parcela y operario.

🛡️

Bitácora de Mantenimiento

Control de plaga, limpieza, riego, fertilización u otro; producto, dosis y operario.

🔬

Resultados de Laboratorio

Código de muestra, cultivo, análisis, parámetro, valor, unidad y laboratorio.

📊

Panel de Investigación

Totales y gráficos de severidad y enfermedades por ensayo.

5 Sincronización (ida y vuelta)

El botón ☁ hace la sincronización completa y al abrir la app se sincroniza sola.

  1. Sube lo pendiente

    Envía a Firestore todo lo que esté marcado como “Pendiente de subida”.

  2. Baja y mezcla lo de la nube

    Trae los registros de Firestore y los combina con lo local sin borrar nunca tus datos del dispositivo. Si existe en ambos lados, se queda con el más reciente.

  3. Recupera lo perdido

    Si limpiaste el almacenamiento local, al abrir la app o tocar ☁ tus registros de la nube vuelven a aparecer.

6 Panel de Investigación

Tarjetas con totales y dos gráficos (fl_chart).

  • Ensayos, Evaluaciones, Aplicaciones, Mantenimiento, Laboratorio: conteos rápidos.
  • Evolución de Severidad (% en el tiempo) para el ensayo seleccionado.
  • Evaluaciones por enfermedad (barras).
  • Usá el selector de ensayo para filtrar los gráficos.

7 Solución de problemas

SíntomaCausa probableSolución
“Firebase no configurado”No inicializó FirebaseVerificá Auth anónimo y que lib/firebase_options.dart exista
“Error: …” al sincronizarFirestore rechaza lectura/escrituraHabilitá Auth anónimo y firebase deploy --only firestore
No veo registros en la nubeAún no sincronicé o falló el uploadTocá ☁ y revisá el mensaje; desplegá las reglas
Datos perdidos localmenteSe limpió el almacenamientoSincronizá para recuperar desde la nube
“Developer Mode” (Windows)Faltan permisos de symlinkUsá -d chrome o activá el Modo Desarrollador de Windows

8 Desarrollo y build automático

Análisis y tests

flutter analyze
flutter test

Build web (resultado en build/web)

flutter build web --release

GitHub Actions — .github/workflows/build.yml corre en cada push/PR a main: pub get → analyze → test → build web y publica build/web como artefacto web-release.
